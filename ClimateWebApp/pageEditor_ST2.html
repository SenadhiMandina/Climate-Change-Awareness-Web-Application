2. Technical Description

Home Page

The home page uses a responsive grid layout and hero section. Key HTML elements include:

  • <section class="hero"> for the main banner
  • <div class="content-tiles"> for showcasing different content areas

CSS techniques include:

  • display: grid with grid-template-columns for responsive layout
  • background-image with overlay for the hero section
  • CSS transitions for hover effects on content tiles

Table Page

The Table Page presents structured climate action data in a responsive and accessible format. Key technical elements include:

HTML Structure
  • <table class="climate-table"> organizes data into rows and columns.
  • <th> elements with <img> icons and title attributes enhance readability and provide tooltips on hover.
  • <td> elements include data values, with icons and progress bars for visualization.
CSS Styling
  • Responsive Design: .table-container { overflow-x: auto; } ensures horizontal scrolling for smaller screens. Alternating row colors (:nth-child(even)) improve readability.
  • Interactive Elements: Hover effects on <th> elements display additional information. The .progress-bar and .progress-value create dynamic progress indicators for climate goals.
  • Accessibility Features: aria-label on the table enhances screen reader navigation. High-contrast colors and tooltips improve usability.

The page efficiently presents climate data while ensuring a seamless user experience across devices.

Sustainable Living Content Page

This page focuses on presenting information clearly and engagingly. Key elements include:

  • <nav class="content-nav"> for in-page navigation
  • <section> tags for each topic area
  • <div class="fact-box"> for highlighting key information

CSS techniques include:

  • position: sticky for the in-page navigation
  • Custom styled list items for easy readability
  • Responsive images using max-width: 100% and height: auto

5. Challenges and Lessons Learned

During the implementation of these pages, several challenges were encountered:

  • Responsive Design: Ensuring the splash screen looked good on all device sizes was challenging. I learned to use media queries and flexbox to create a responsive layout.
  • Accessibility: Making the volunteer form accessible required careful consideration of form labels and ARIA attributes. This improved my understanding of web accessibility principles.
  • Content Organization: Structuring the climate science content in a clear and engaging way was difficult. I learned the importance of information architecture and how to use HTML semantics to improve content structure.

These challenges taught me the importance of planning, testing on multiple devices, and considering accessibility from the start of the design process.

6. Accessibility

Accessibility was a key consideration in the development of these pages:

  • Semantic HTML: Used throughout to provide clear structure and meaning to content.
  • Color Contrast: Ensured sufficient contrast between text and background colors.
  • Alternative Text: Provided for all images, including descriptive captions where necessary.
  • Keyboard Navigation: All interactive elements are accessible via keyboard.
  • ARIA Labels: Used where necessary to provide additional context for screen readers.
  • Responsive Design: Ensures content is accessible on various device sizes and orientations.

7. Compliance

The pages created comply with Janet regulations for web page publication in the following ways:

  • Data Protection: No personal data is collected or stored without explicit consent.
  • Copyright: All images and content used are either original, properly licensed, or used with permission.
  • Accessibility: Pages are designed to meet WCAG 2.1 Level AA standards.
  • Security: No sensitive information is exposed in the HTML or CSS.
  • Branding: University branding guidelines have been followed in the design.

While efforts have been made to ensure compliance, further review by the university's web team would be necessary for full verification.
